# Core Engine — Phase 1 Foundation (MML-B-0001)

Implements the framework-agnostic `@mindmaplib/core` engine per spec MML-B-0001 (v0.5.0, accepted). This PR delivers the full Phase 1 foundation plus the monorepo scaffolding required to build, test, and publish it.

## What landed

### Monorepo scaffolding

- pnpm workspace (`packages/*`, `demo`), root scripts
- `packages/core`: package.json, tsconfig (strict, no DOM lib), tsup build, index
- Tooling: prettier, eslint (typescript-eslint flat config, `no-explicit-any`), vitest (v8 coverage, 80% threshold), dependency-cruiser (core isolation rules), changesets, tsup
- CI: `ci.yml` (format/lint/typecheck/test+coverage/boundaries, node 22), `release.yml` (changesets publish), CODEOWNERS
- `.nvmrc` pins node 22 LTS (dep-cruiser requires ^22||^24||>=26)

### Core engine (13 modules, 103 tests)

- `types.ts` — MindmapDoc, MindmapNode, NodeContent (ProseMirror subset), TransactionOp discriminated union, EditorState, store/serialization types
- `errors.ts` — MindmapError (9 codes), VersionConflictError, StoreError
- `id.ts` — dependency-free unique id (crypto.randomUUID + fallback)
- `content.ts` — emptyContent, normalizeContent (allowed types/attrs, max list depth 4, max text 10k, URL scheme allowlist, no DOM lib)
- `transactions.ts` — op factories, buildTransaction, applyOp (pure structural transform), applyTransaction (version bump + baseVersion strict check)
- `document.ts` — createDoc, queries (getNode/getChildren/getDescendants/getPath/getAncestors), convenience ops with immutability + structural sharing
- `validation.ts` — validateDoc (all 11 document invariants)
- `serialize.ts` — serialize/deserialize with schemaVersion=1, unknown-field stripping
- `store.ts` — InMemoryStore default implementation
- `layout.ts` — computeLayoutOps via d3-hierarchy (tree-h/v/radial, manual anchors, collapsed-as-leaf)
- `editor.ts` — MindmapEditor stateful controller (undo/redo ring buffer, selection, viewport, subscribe, store integration)
